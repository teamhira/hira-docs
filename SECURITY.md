# Security Policy

## Scope
This repository contains documentation. While it does not house primary production code, security issues must still be handled seriously, especially regarding:
- Accidental documentation of sensitive information.
- Configuration details with potential for misuse.
- Exposure of endpoints or internal flows that should not be public.

## How to Report
- Do not open public issues for sensitive details.
- Report directly to the internal maintainers or security contact.
- Include context, impact, and mitigation recommendations if available.

## Response SLA
- Initial Acknowledgement: Maximum 2 business days.
- Triage: Maximum 5 business days.
