# Review Process

This document outlines the mandatory review process for documentation changes.

## Review Criteria
- **Accuracy**: Does the document reflect the current state of the architecture or product?
- **Clarity**: Is the language clear and professional?
- **SSOT Alignment**: Does it align with existing "Single Source of Truth" definitions?
- **Cross-document Impact**: Have related documents been updated?

## Procedure
1. Create a Pull Request with a clear description.
2. Tag the relevant area owners (see `CODEOWNERS`).
3. Address any feedback provided by the reviewers.
4. Update `CHANGELOG.md` if the changes have a broad impact across the repository.
5. Merge once approved.
